  2. Surface Analysis

    The purpose of this lab is to familiarize yourself with the tools in the Surface analysis tool set.

    The data folder contains a personal geodatabase named SurfaceDatabase.mdb with the following datasets:

    • A DEM created by merging Two ASTER GDEM 2 Rasters from Earth Explorer (ASTGTM2_N36W119 & ASTGTM@_37N119W) and clipping the rasters to the Trail layer (included) to improve processing time.
    • A selection of trails created from a layer of trails I obtained from Kings Canyon National Park.
    • SEKIHydro: A dataset of water features clipped to the Sierra Raster. The original Dataset was provided by Kings Canyon National Park.
    • HydroQuartMi: a raster displaying a mile buffer around all water features in the SEKIHydro Layer. This layer was created using the Buffer tool and the Polygon to Raster tool.
    • Hydro100: a raster displaying a 100 ft buffer around all water features in the SEKIHydro Layer. This layer was created using the Buffer tool and the Polygon to Raster tool.
    • TrailBuff: a raster displaying a mile buffer around the Trail feature dataset.. This layer was created using the Buffer tool and the Polygon to Raster tool.

    Scenario

    You are planning your dream camping and backpacking trip through Californias Sierra Nevada Mountains, Kings Canyon and Sequoia National Parks. Before you hit the trail, you want to use your GIS skills to find campsites along the trail that meet the following criteria.

    Kings Canyon Camping Rules

    • Campsites must be more than 100' from water
    • Camp Fires are not permitted above 10,000'

    Your Desired Criteria

    • Less than Mile from water
    • Less than Mile from the Trail
    • Southern or Eastern Exposure
    • Area at least 100sq Meters in size less than 3 degree slope
    • Good View

    To achieve your Goal of finding the perfect camp sites, you will use the following Surface Analysis tools.

    • Slope: used to find the degree of slope or percent rise over run of an area, this tool will be used to find a flatter camp site.
    • Aspect: Used to find the downhill direction of a slope. You will use this tool to find camp sites that have a good southern or eastern exposure, to provide warm early morning light.
    • Viewshed: The viewshed tool will help determine what areas you will be able to see from your selected camp sites, the larger the viewshed the better the better the scenery.
    • Contour: Contour lines will help you determine how steep the trail is along your route.

    13. What does the above expression mean? What information did it yield? Can you interpret these cell values?

      The above expression is essentially a giant spatial criteria which filters out all data except all the criteria we are interested in - an "intersection" critieria. The AspectSierra criteria ensures that only slopes facing 67 to 202 degrees are included; the SlopeSierra criteria ensures that only sites with a 3 or less slope are chosen; and the Sierra criteria includes sites that are at or less than 10,000' to ensure we can have a campfire. The Raster Calculator was supposed to give us all cells in the raster that matched our criteria. Unfortunately it gave us cells with western facing slopes instead of eastern ones. Brown is the cells that are 100' away from water, Green cells are SE facing slopes, Purple cells are west facing slopes, and Orange-red cells match all criteria.

  3. Suitability Analysis

    The Weighted Overlay Analysis tool has many applications, from finding the optimal habitat for preservation purposes for a specific endangered species to finding the optimal site for a new store. This technique for applying a common scale of values to diverse and dissimilar inputs to create an integrated analysis is called suitability analysis. The following articles will introduce you to this concept:
    • The Tool
    • Understanding it

You are looking for a place to live, and you consider the factors for which you have spatial data. You know all of these factors are important to you, but you value some factors more than others.   4. Weighted Overlay Analysis

    The weighted overlay tool (and how it works) overlays several rasters using a common measurement scale and weights each according to its importance. We know that near locations are more important that far locations, so we will assign the values accordingly. We will weigh each input equally, but the tool can weigh each input differently. Since the tool requires all inputs to be integer rasters, we will need to reclassify our rasters to integer rasters. We will also use this opportunity to convert the mile buffer values to a scale from one to ten, where ten is our most valuable location.

    1. Open the ArcToolbox | Spatial Analyst Tools | Reclass | Reclassify  tool.
    2. Convert each raster according to the inputs and outputs:



      Python: 
      import arcpy  
from arcpy import env
from arcpy.sa import *
arcpy.env.workspace = "X:/Hostgator/homework/GSP537/labs/2/data/Suitability.gdb"
objClasses = ["DowntownPhoenix_MultipleRing_raster", "Canal_MultipleRing_clip_raster", "LightRailStations_MultipleRing_clip_raster", 
              "Street_MultipleRing_clip_raster"]
lsRemap = RemapValue([[0.25, 10], [0.5, 7],[1, 3], [5, 1]])
for objClass in objClasses:
  outReclassify = Reclassify(objClass, "VALUE", lsRemap, "NODATA")
  outReclassify.save(objClass + "_IntegerReclassify")
    3. Repeat for each raster. Note that 5 values for streets will not exist because the maximum distance between streets is 1 mile. Also note, if you use python to do this, that the Reclassify function will not add the resulting rasters to the table of contents; if you want them visible there (not required), you will have to add them yourself.
